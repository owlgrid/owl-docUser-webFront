# Rights

Les droits sont l'association d'une action sur un périmètre donné. Pour qu'un droit puisse être utilisé, il doit être assigné à un utilisateur.

## Caractéristiques des droits

Un droit est identifié de la manière suivante : `perimeter.action`. Pour savoir comment un périmètre est construit, cf. [Périmètres](/docs/general/concepts/perimeters). Pour savoir comment une action est construite, cf. [Actions](actions).

Le droit le plus élevé dans la hiérarchie est `owl.admin`. Il est automatiquement assigné au compte administrateur et permet d'instancier OwlGrid et de créer de nouveaux droits.

## Assignation des droits

Pour qu'un droit puisse être utilisé, il doit être associé à un [utilisateur](users) ou à un [groupe](groups). Attention, lorsqu'un droit est assigné à un groupe, il est automatiquement assigné à tous les utilisateurs qui sont membres de ce groupe. De plus, lorsqu'un droit est assigné à un groupe parent, tous ses groupes enfants en héritent (cf. [Groupes](groups)).
