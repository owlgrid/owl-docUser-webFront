# Hosting OwlGrid

## Introduction

OwlGrid offers two ways of running: as a self-hosted server or as a cloud service.

## Cloud

OwlGrid Cloud is the easiest way of running OwlGrid without having to concern about ressources, performances and maintenance. It is also the most cost-effective way of running OwlGrid.

### Hosting options

#### Cloud provider

OwlGrid Cloud can be runned on several cloud provider. For now, only AWS is supported but we plan to add more in the future.

#### Region

OwlGrid Cloud can be runned on several regions, depending on the cloud provider. For now, only AWS regions are supported but we plan to add more in the future.

:::info

Cross-regions instances are not available yet.

:::

### Get started

To start using OwlGrid Cloud, you can sign up [here](https://owlgrid.app/signup) and start using OwlGrid right away.

You can also contact us at [hello@owlgrid.com](mailto:hello@owlgrid.com).

## Self-hosted

OwlGrid can also be self-hosted on your own infrastructure. This is the most flexible way of running OwlGrid, but also the most complex one.

Refer to the OwlGrid Github repositories to get started: https://github.com/owlgrid.
